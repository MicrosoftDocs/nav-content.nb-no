# <a name="design-details-balancing-supply-with-demand"></a><span data-ttu-id="fa18d-104">Designdetaljer: Balansere forsyning med behov</span><span class="sxs-lookup"><span data-stu-id="fa18d-104">Design Details: Balancing Supply with Demand</span></span>
<span data-ttu-id="fa18d-105">Kjernen i planleggingssystemet omfatter balansering av behov og forsyning ved å foreslå brukerhandlinger som endrer forsyningsordrene hvis det oppstår ubalanse.</span><span class="sxs-lookup"><span data-stu-id="fa18d-105">The core of the planning system involves balancing demand and supply by means of suggesting user actions to revise the supply orders in case of imbalance.</span></span> <span data-ttu-id="fa18d-106">Dette skjer per kombinasjon av variant og lokasjon.</span><span class="sxs-lookup"><span data-stu-id="fa18d-106">This takes place per combination of variant and location.</span></span>  
  
<span data-ttu-id="fa18d-107">Tenk deg at hver lagerprofil inneholder en streng med behovshendelser (sortert etter dato og prioritet) og en tilsvarende streng for forsyningshendelser.</span><span class="sxs-lookup"><span data-stu-id="fa18d-107">Imagine that each inventory profile contains a string of demand events (sorted by date and priority) and a corresponding string of supply events.</span></span> <span data-ttu-id="fa18d-108">Hver hendelse refererer tilbake til kildetype og identifikasjon.</span><span class="sxs-lookup"><span data-stu-id="fa18d-108">Each event refers back to its source type and identification.</span></span> <span data-ttu-id="fa18d-109">Reglene for utligning av varen er enkle.</span><span class="sxs-lookup"><span data-stu-id="fa18d-109">The rules for counterbalancing the item are straightforward.</span></span> <span data-ttu-id="fa18d-110">Fire forekomster av tilsvarende behov og forsyning kan oppstå når som helst i prosessen:</span><span class="sxs-lookup"><span data-stu-id="fa18d-110">Four instances of matching demand and supply can occur at any point of time in the process:</span></span>  
  
1. <span data-ttu-id="fa18d-111">Det finnes verken behov eller forsyning for varen = > planleggingen er ferdig (eller skal ikke starte).</span><span class="sxs-lookup"><span data-stu-id="fa18d-111">No demand or supply exists for the item => the planning has finished (or should not start).</span></span>  
2. <span data-ttu-id="fa18d-112">Det finnes behov, men ingen forsyning = > forsyning må foreslås.</span><span class="sxs-lookup"><span data-stu-id="fa18d-112">Demand exists but there is no supply => supply should be suggested.</span></span>  
3. <span data-ttu-id="fa18d-113">Det finnes forsyning, men ikke behov for det = > forsyning må avbrytes.</span><span class="sxs-lookup"><span data-stu-id="fa18d-113">Supply exists but there is no demand for it => supply should be canceled.</span></span>  
4. <span data-ttu-id="fa18d-114">Både behov og forsyning finnes => spørsmål må stilles og besvares før systemet kan sikre at behovet kan dekkes og forsyningen er tilstrekkelig.</span><span class="sxs-lookup"><span data-stu-id="fa18d-114">Both demand and supply exist => questions should be asked and answered before the system can ensure that demand will be met and supply is sufficient.</span></span>  
  
     <span data-ttu-id="fa18d-115">Hvis tidspunktet for forsyningen ikke passer, kan kanskje forsyningen planlegges på nytt som følger:</span><span class="sxs-lookup"><span data-stu-id="fa18d-115">If the timing of the supply is not suitable, perhaps the supply can be rescheduled as follows:</span></span>  
  
    1.  <span data-ttu-id="fa18d-116">Hvis forsyningen plasseres tidligere enn behovet, kan kanskje forsyningen planlegges ut slik at beholdningen er så lavt som mulig.</span><span class="sxs-lookup"><span data-stu-id="fa18d-116">If the supply is placed earlier than the demand, perhaps the supply can be rescheduled out so that inventory is as low as possible.</span></span>  
    2.  <span data-ttu-id="fa18d-117">Hvis forsyningen plasseres senere enn behovet, kan kanskje forsyningen planlegges inn.</span><span class="sxs-lookup"><span data-stu-id="fa18d-117">If the supply is placed later than the demand, perhaps the supply can be rescheduled in.</span></span> <span data-ttu-id="fa18d-118">Hvis ikke vil systemet foreslå ny forsyning.</span><span class="sxs-lookup"><span data-stu-id="fa18d-118">Otherwise, the system will suggest new supply.</span></span>  
    3.  <span data-ttu-id="fa18d-119">Hvis forsyningen oppfyller behovet på datoen, kan planleggingssystemet fortsette å undersøke om forsyningsantallet kan dekke behovet.</span><span class="sxs-lookup"><span data-stu-id="fa18d-119">If the supply meets the demand on the date, the planning system can proceed to investigate whether the quantity of the supply can cover the demand.</span></span>  
  
     <span data-ttu-id="fa18d-120">Når tidsberegningen er på plass, kan det tilstrekkelig antallet som skal leveres, beregnes slik:</span><span class="sxs-lookup"><span data-stu-id="fa18d-120">Once the timing is in place, the adequate quantity to be supplied can be calculated as follows:</span></span>  
  
    1.  <span data-ttu-id="fa18d-121">Hvis forsyningsantallet er mindre enn behovet, kan det hende at forsyningsantallet kan reduseres (eller ikke, hvis det er begrenset av et prinsipp for maksimumsantall).</span><span class="sxs-lookup"><span data-stu-id="fa18d-121">If the supply quantity is less than the demand, it is possible that the supply quantity could be increased (or not, if limited by a maximum quantity policy).</span></span>  
    2.  <span data-ttu-id="fa18d-122">Hvis forsyningsantallet er større enn behovet, kan det hende at forsyningsantallet kan reduseres (eller ikke, hvis det er begrenset av et prinsipp for minimumsantall).</span><span class="sxs-lookup"><span data-stu-id="fa18d-122">If the supply quantity is greater than the demand, it is possible that the supply quantity can be decreased (or not, if limited by a minimum quantity policy).</span></span>  
  
     <span data-ttu-id="fa18d-123">På dette tidspunktet finnes en av disse to situasjonene:</span><span class="sxs-lookup"><span data-stu-id="fa18d-123">At this point, either of these two situations exists:</span></span>  
  
    1.  <span data-ttu-id="fa18d-124">Gjeldende behov kan dekkes, og i så fall kan det lukkes, og planlegging for neste behov kan starte.</span><span class="sxs-lookup"><span data-stu-id="fa18d-124">The current demand can be covered, in which case it can be closed and planning for the next demand can start.</span></span>  
    2.  <span data-ttu-id="fa18d-125">Forsyningen har nådd maksimumet, og noe av behovsmengden er ikke dekket.</span><span class="sxs-lookup"><span data-stu-id="fa18d-125">The supply has reached its maximum, leaving some of the demand quantity uncovered.</span></span> <span data-ttu-id="fa18d-126">I slike tilfeller kan planleggingssystemet lukke gjeldende forsyningen og fortsette til den neste.</span><span class="sxs-lookup"><span data-stu-id="fa18d-126">In this case, the planning system can close the current supply and proceed to the next one.</span></span>  
  
 <span data-ttu-id="fa18d-127">Prosessen starter helt på nytt med det neste behovet og den aktuelle forsyningen eller omvendt.</span><span class="sxs-lookup"><span data-stu-id="fa18d-127">The procedure starts all over with the next demand and the current supply or vice versa.</span></span> <span data-ttu-id="fa18d-128">Gjeldende forsyning kan kanskje dekke neste behov også, eller gjeldende behov er ikke fullstendig dekket ennå.</span><span class="sxs-lookup"><span data-stu-id="fa18d-128">The current supply might be able to cover this next demand as well, or the current demand has not yet been fully covered.</span></span>  
  
## <a name="rules-concerning-actions-for-supply-events"></a><span data-ttu-id="fa18d-129">Regler vedrørende handlinger for forsyningshendelser</span><span class="sxs-lookup"><span data-stu-id="fa18d-129">Rules Concerning Actions for Supply Events</span></span>  
<span data-ttu-id="fa18d-130">Når planleggingssystemet utfører en beregning ovenfra og nedover der forsyning må dekke behov, tas behovet som gitt, det vil si at planleggingssystemet ikke kan kontrollere det.</span><span class="sxs-lookup"><span data-stu-id="fa18d-130">When the planning system performs a top-down calculation in which supply must fulfill demand, the demand is taken as a given, that is, it lies outside the control of the planning system.</span></span> <span data-ttu-id="fa18d-131">Forsyningssiden kan Imidlertid administreres.</span><span class="sxs-lookup"><span data-stu-id="fa18d-131">However, the supply side can be managed.</span></span> <span data-ttu-id="fa18d-132">Planleggingssystemet foreslår derfor å opprette nye forsyningsordrer, tidsplanlegge eksisterende på nytt og/eller endre ordreantallet.</span><span class="sxs-lookup"><span data-stu-id="fa18d-132">Therefore, the planning system will suggest creating new supply orders, rescheduling existing ones, and/or changing the order quantity.</span></span> <span data-ttu-id="fa18d-133">Hvis en eksisterende forsyningsordre blir overflødige foreslår planleggingssystemet at brukeren avbryter den.</span><span class="sxs-lookup"><span data-stu-id="fa18d-133">If an existing supply order becoming superfluous, the planning system will suggest that the user cancels it.</span></span>  
  
<span data-ttu-id="fa18d-134">Hvis brukeren vil utelate en eksisterende forsyningsordre fra planleggingsforslagene, kan det angis at den ikke har planleggingsfleksibilitet (Planleggingsfleksibilitet = ingen).</span><span class="sxs-lookup"><span data-stu-id="fa18d-134">If the user wants to exclude an existing supply order from the planning suggestions, he can state that it has no planning flexibility (Planning Flexibility = None).</span></span> <span data-ttu-id="fa18d-135">Overflødig forsyning fra denne ordren brukes deretter til å dekke behovet, men ingen handling foreslås.</span><span class="sxs-lookup"><span data-stu-id="fa18d-135">Then, excess supply from that order will be used to cover demand, but no action will be suggested.</span></span>  
  
<span data-ttu-id="fa18d-136">All forsyning har vanligvis en planleggingsfleksibilitet som er begrenset av betingelsene i hver av de foreslåtte handlingene.</span><span class="sxs-lookup"><span data-stu-id="fa18d-136">In general, all supply has a planning flexibility that is limited by the conditions of each of the suggested actions.</span></span>  
  
-   <span data-ttu-id="fa18d-137">**Tidsplanlegg ut på nytt**: Datoen for en eksisterende forsyningsordre kan tidsplanlegges ut for å oppfylle forfallsdatoen for behovet med mindre følgende gjelder:</span><span class="sxs-lookup"><span data-stu-id="fa18d-137">**Reschedule Out**: The date of an existing supply order can be scheduled out to meet the demand due date unless:</span></span>  
  
    -   <span data-ttu-id="fa18d-138">Den representerer beholdning (alltid på dag null).</span><span class="sxs-lookup"><span data-stu-id="fa18d-138">It represents inventory (always on day zero).</span></span>  
    -   <span data-ttu-id="fa18d-139">Den har en ordre-til-ordre som er koblet til er annet behov.</span><span class="sxs-lookup"><span data-stu-id="fa18d-139">It has an order-to-order linked to another demand.</span></span>  
    -   <span data-ttu-id="fa18d-140">Dette ligger utenfor vinduet for å planlegge på nytt, som angitt av tidsperioden.</span><span class="sxs-lookup"><span data-stu-id="fa18d-140">It lies outside the reschedule window defined by the time bucket.</span></span>  
    -   <span data-ttu-id="fa18d-141">En forsyning som er nærmere, kan brukes.</span><span class="sxs-lookup"><span data-stu-id="fa18d-141">There is a closer supply that could be used.</span></span>  
    -   <span data-ttu-id="fa18d-142">På den annen side kan brukeren velge ikke å planlegge på nytt fordi:</span><span class="sxs-lookup"><span data-stu-id="fa18d-142">On the other hand, the user may decide not to reschedule because:</span></span>  
    -   <span data-ttu-id="fa18d-143">Forsyningsordren er allerede knyttet til et annet behov på en tidligere dato.</span><span class="sxs-lookup"><span data-stu-id="fa18d-143">The supply order has already been tied to another demand on a previous date.</span></span>  
    -   <span data-ttu-id="fa18d-144">Den nye planleggingen som kreves, er så minimal at brukeren synes den er ubetydelig.</span><span class="sxs-lookup"><span data-stu-id="fa18d-144">The needed rescheduling is so minimal that the user finds it negligible.</span></span>  
  
-   <span data-ttu-id="fa18d-145">**Tidsplanlegg inn på nytt**: datoen for en eksisterende forsyningsordre kan tidsplanlegges inn, unntatt når følgende gjelder:</span><span class="sxs-lookup"><span data-stu-id="fa18d-145">**Reschedule In**: The date of an existing supply order can be scheduled in, except in the following conditions:</span></span>  
  
    -   <span data-ttu-id="fa18d-146">Den er koblet direkte til et annet behov.</span><span class="sxs-lookup"><span data-stu-id="fa18d-146">It is linked directly to some other demand.</span></span>  
    -   <span data-ttu-id="fa18d-147">Dette ligger utenfor vinduet for å planlegge på nytt, som angitt av tidsperioden.</span><span class="sxs-lookup"><span data-stu-id="fa18d-147">It lies outside the reschedule window defined by the time bucket.</span></span>  
  
> [!NOTE]  
>  <span data-ttu-id="fa18d-148">Når en vare planlegges ved hjelp av et gjenbestillingspunkt, kan forsyningsordren alltid om nødvendig tidsplanlegges inn.</span><span class="sxs-lookup"><span data-stu-id="fa18d-148">When planning an item using a reorder point, the supply order can always be scheduled in if necessary.</span></span> <span data-ttu-id="fa18d-149">Dette er vanlig i fremoverplanlagte forsyningsordrer som utløses av et gjenbestillingspunkt.</span><span class="sxs-lookup"><span data-stu-id="fa18d-149">This is common in forward-scheduled supply orders triggered by a reorder point.</span></span>  
  
-   <span data-ttu-id="fa18d-150">**Øk antall**: Antallet i en eksisterende forsyningsordre kan økes for å dekke behovet, med mindre forsyningsordren er koblet direkte til et behov med en ordre-til-ordre-kobling.</span><span class="sxs-lookup"><span data-stu-id="fa18d-150">**Increase Quantity**: The quantity of an existing supply order can be increased to meet the demand unless the supply order is linked directly to a demand by an order-to-order link.</span></span>  
  
> [!NOTE]  
>  <span data-ttu-id="fa18d-151">Selv om det er mulig å øke forsyningsordren, kan det være begrenset på grunn av et maksimumsordreantall.</span><span class="sxs-lookup"><span data-stu-id="fa18d-151">Even though it is possible to increase the supply order, it may be limited due to a defined maximum order quantity.</span></span>  
  
-   <span data-ttu-id="fa18d-152">**Reduser antall**: En eksisterende forsyningsordre med et overskudd sammenlignet med et eksisterende behov kan reduseres for å dekke behovet.</span><span class="sxs-lookup"><span data-stu-id="fa18d-152">**Decrease Quantity**: An existing supply order with a surplus compared to an existing demand can be decreased to meet the demand.</span></span>  
  
> [!NOTE]  
>  <span data-ttu-id="fa18d-153">Selv om antallet kan reduseres, kan det fortsatt være noen overskudd sammenlignet med behovet på grunn av et angitt minimumsordreantall eller en bestillingsfaktor.</span><span class="sxs-lookup"><span data-stu-id="fa18d-153">Even though the quantity could be decreased, there may still be some surplus compared to the demand due to a defined minimum order quantity or order multiple.</span></span>  
  
-   <span data-ttu-id="fa18d-154">**Avbryt**: Forsyningsordren kan avbrytes som en spesiell hendelse for handlingen for å redusere antallet, hvis den er blitt redusert til null.</span><span class="sxs-lookup"><span data-stu-id="fa18d-154">**Cancel**: As a special incident of the decrease quantity action, the supply order could be canceled if it has been decreased to zero.</span></span>  
-   <span data-ttu-id="fa18d-155">**Ny**: Hvis det ikke allerede finnes en forsyningsordre eller en eksisterende ikke kan endres til å dekke det nødvendige antallet på den etterspurte forfallsdatoen, blir det foreslått en ny forsyningsordre.</span><span class="sxs-lookup"><span data-stu-id="fa18d-155">**New**: If no supply order already exists, or an existing one cannot be changed to meet the necessary quantity on the demanded due date, a new supply order is suggested.</span></span>  
  
## <a name="determining-the-supply-quantity"></a><span data-ttu-id="fa18d-156">Fastslå forsyningsantall</span><span class="sxs-lookup"><span data-stu-id="fa18d-156">Determining the Supply Quantity</span></span>  
<span data-ttu-id="fa18d-157">Planleggingsparametre angitt av brukeren, styrer foreslåtte antall for hver ordre.</span><span class="sxs-lookup"><span data-stu-id="fa18d-157">Planning parameters defined by the user control the suggested quantity of each supply order.</span></span>  
  
<span data-ttu-id="fa18d-158">Når planleggingssystemet beregner antallet for en ny forsyningsordre eller det endrede antallet i en eksisterende, kan foreslåtte antallet være forskjellig fra antallet det faktisk er behov for.</span><span class="sxs-lookup"><span data-stu-id="fa18d-158">When the planning system calculates the quantity of a new supply order or the quantity change on an existing one, the suggested quantity may be different from what is actually demanded.</span></span>  
  
<span data-ttu-id="fa18d-159">Hvis maksimumsbeholdning eller fast ordreantall som er valgt, kan det foreslåtte vareantallet økes for å oppfylle det faste antallet eller maksimumsbeholdningen.</span><span class="sxs-lookup"><span data-stu-id="fa18d-159">If a maximum inventory or fixed order quantity are selected, the suggested quantity may be increased to meet that fixed quantity or the maximum inventory.</span></span> <span data-ttu-id="fa18d-160">Hvis gjenbestillingsprinsippet bruker et gjenbestillingspunkt, kan antallet i det minste økes for å dekke gjenbestillingspunktet.</span><span class="sxs-lookup"><span data-stu-id="fa18d-160">If a reordering policy uses a reorder point, the quantity may be increased at least to meet the reorder point.</span></span>  
  
 <span data-ttu-id="fa18d-161">Du kan endre det foreslåtte antallet i denne rekkefølgen:</span><span class="sxs-lookup"><span data-stu-id="fa18d-161">The suggested quantity may be modified in this sequence:</span></span>  
  
1. <span data-ttu-id="fa18d-162">Ned til maks maksimumsordreantallet (hvis det finnes).</span><span class="sxs-lookup"><span data-stu-id="fa18d-162">Down to the maximum order quantity (if any).</span></span>  
2. <span data-ttu-id="fa18d-163">Opp til det minste bestillingsantallet.</span><span class="sxs-lookup"><span data-stu-id="fa18d-163">Up to the minimum order quantity.</span></span>  
3. <span data-ttu-id="fa18d-164">Opp for oppfylle nærmeste bestillingsfaktor.</span><span class="sxs-lookup"><span data-stu-id="fa18d-164">Up to meet the nearest order multiple.</span></span> <span data-ttu-id="fa18d-165">(Dette kan bryte med maksimumsordreantallet ved feile innstillinger.)</span><span class="sxs-lookup"><span data-stu-id="fa18d-165">(In case of erroneous settings, this may violate the maximum order quantity.)</span></span>  
  
## <a name="order-tracking-links-during-planning"></a><span data-ttu-id="fa18d-166">Sporingskoblinger under planlegging</span><span class="sxs-lookup"><span data-stu-id="fa18d-166">Order Tracking Links during Planning</span></span>  
<span data-ttu-id="fa18d-167">Når det gjelder sporing under planleggingen, er det viktig å nevne at planleggingssystemet ordner de dynamisk opprettede sporingskoblingene for vare-/variant-/lokasjonskombinasjoner.</span><span class="sxs-lookup"><span data-stu-id="fa18d-167">Concerning order tracking during planning, it is important to mention that the planning system rearranges the dynamically created order tracking links for the item/variant/location combinations.</span></span>  
  
<span data-ttu-id="fa18d-168">Det er to årsaker til dette:</span><span class="sxs-lookup"><span data-stu-id="fa18d-168">There are two reasons for this:</span></span>  
  
-   <span data-ttu-id="fa18d-169">Planleggingssystemet må kunne justere forslagene, slik at alle behov dekkes, og at ingen forsyningsordrer er overflødige.</span><span class="sxs-lookup"><span data-stu-id="fa18d-169">The planning system must be able to justify its suggestions; that all demand has been covered, and that no supply orders are superfluous.</span></span>  
-   <span data-ttu-id="fa18d-170">Sporingskoblinger som er opprettet dynamisk må balanseres på nytt regelmessig.</span><span class="sxs-lookup"><span data-stu-id="fa18d-170">Dynamically created order tracking links need to be rebalanced regularly.</span></span>  
  
<span data-ttu-id="fa18d-171">Over tid kommer dynamiske sporingskoblinger ut av balanse fordi hele ordrensporingsnettverket ikke endres på nytt før en behovs- eller forsyningshendelse faktisk er lukket.</span><span class="sxs-lookup"><span data-stu-id="fa18d-171">Over time, dynamic order tracking links become out of balance since the entire order tracking network is not rearranged until a demand or supply event is actually closed.</span></span>  
  
<span data-ttu-id="fa18d-172">Før du balansere forsyningen ved behov, slettes alle eksisterende sporingskoblinger.</span><span class="sxs-lookup"><span data-stu-id="fa18d-172">Before balancing supply by demand, the program deletes all existing order tracking links.</span></span> <span data-ttu-id="fa18d-173">Når en en behovs- eller forsyningshendelse deretter lukkes under balanseringsprosessen, oppretter det nye sporingskoblinger mellom behov og forsyning.</span><span class="sxs-lookup"><span data-stu-id="fa18d-173">Then during the balancing procedure, when a demand or supply event is closed, it establishes new order tracking links between the demand and supply.</span></span>  
  
> [!NOTE]  
>  <span data-ttu-id="fa18d-174">Selv om varen ikke er konfigurert for dynamisk ordresporing, vil det planlagte systemet opprette balanserte sporingskoblinger, som beskrevet ovenfor.</span><span class="sxs-lookup"><span data-stu-id="fa18d-174">Even if the item is not set up for dynamic order tracking, the planned system will create balanced order tracking links as explained above.</span></span>
